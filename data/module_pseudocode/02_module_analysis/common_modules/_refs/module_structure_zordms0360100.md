### 3.2 a000_input_validation (입력 검증)

**설명**: 입력 데이터의 유효성을 검증

**처리 로직**:
1. 초기값 설정 (`p010_set_initial_val`)
2. 필수 항목 체크 (`p100_chk_mndt_item`)
3. 선택 항목 체크 및 기본값 설정 (`p200_chk_mtcg_item`)

**주요 검증 항목**:
- 서비스관리번호 (필수)
- 상품ID (필수)
- 처리일자 (선택, 기본값: 현재일자)

---

### 3.3 c000_상품공통기능체크 (상품 공통 체크)

**설명**: 모든 상품에 공통으로 적용되는 기본 체크 로직

**입력 파라미터**:
- 상품 객체 (상품ID, 상품구분코드 등)

**주요 처리 로직**:

1. **상품 기본정보 조회**
   - 상품마스터 테이블에서 상품 정보 조회
   - 상품 미존재 시 처리 불가

2. **판매 가능 여부 체크**
   - 판매가능여부 = "N" → 처리 불가

3. **판매 기간 체크**
   - 처리일자가 판매시작일 ~ 판매종료일 범위 내인지 확인

4. **가입 대상 체크**
   - 최소나이 조건 확인
   - 최대나이 조건 확인

5. **채널 제한 체크**
   - 판매채널제한 = "Y"인 경우
   - 현재 채널이 허용채널목록에 포함되는지 확인

6. **중복 가입 체크**
   - 중복가입불가 = "Y"인 경우
   - 동일 상품 가입 이력 확인

7. **재고 체크**
   - 재고관리 = "Y"인 경우
   - 재고수량 > 0 확인

8. **블랙리스트 체크**
   - 고객의 블랙리스트 여부 확인

**반환값**:
- 출력.처리가능여부: "Y" / "N"
- 출력.불가사유: 처리 불가 시 사유

**에러 처리**:
- 각 체크 단계에서 조건 미충족 시 즉시 실패 반환
- 불가사유를 명확히 설정하여 반환

---

### 3.4 요금제_공통체크 (G1)

**설명**: 요금제 상품에 특화된 체크 로직

**주요 처리 로직**:

1. **기존 요금제 조회**
   - 고객의 현재 가입 요금제 확인

2. **최소 사용 기간 체크**
   - 기존 요금제의 최소사용기간 확인
   - 사용기간 < 최소사용기간 → 처리 불가

3. **요금제 변경 제한 체크**
   - 요금제변경제한 테이블 조회
   - 특정 요금제 간 변경 불가 관계 확인

4. **세대 제한 체크**
   - 3G → 5G 직접 변경 불가 (4G 경유 필요)

5. **약정 체크**
   - 유효한 약정 존재 시 위약금 계산
   - 위약금 정보를 출력에 포함

**반환값**:
- 출력.처리가능여부: "Y" / "N"
- 출력.위약금: 발생하는 위약금 금액
- 출력.경고메시지: 약정 위약금 발생 안내

---

### 3.5 부가서비스_공통체크 (I2)

**설명**: 부가서비스 상품에 특화된 체크 로직

**주요 처리 로직**:

1. **기준 요금제 체크**
   - 기준요금제필수 = "Y"인 경우
   - 현재 요금제가 기준요금제목록에 포함되는지 확인

2. **선행 서비스 체크**
   - 선행서비스필수 = "Y"인 경우
   - 모든 선행 서비스의 가입 여부 확인

3. **배타 관계 체크**
   - 배타서비스목록 조회
   - 배타 관계 서비스 가입 중인지 확인

4. **단말 제한 체크**
   - 단말제한 = "Y"인 경우
   - 현재 단말이 허용단말목록에 포함되는지 확인

**반환값**:
- 출력.처리가능여부: "Y" / "N"
- 출력.불가사유: 처리 불가 시 구체적 사유

---

### 3.6 할인_공통체크 (D1)

**설명**: 할인 상품에 특화된 체크 로직

**주요 처리 로직**:

1. **할인 대상 요금제 체크**
   - 현재 요금제가 할인대상요금제목록에 포함되는지 확인

2. **중복 할인 체크**
   - 중복할인불가 = "Y"인 경우
   - 동일 할인유형의 할인 가입 여부 확인

3. **VIP 등급 체크**
   - VIP전용 = "Y"인 경우
   - 고객의 VIP등급이 "VIP" 또는 "VVIP"인지 확인

**반환값**:
- 출력.처리가능여부: "Y" / "N"
- 출력.불가사유: 처리 불가 시 사유

## 7. 특이사항 및 주의사항

### 7.1 처리 특성
- **순차 처리**: 상품 목록을 순회하며 각각 독립적으로 체크
- **조기 종료**: 각 체크 단계에서 조건 미충족 시 즉시 실패 반환
- **상품구분별 분기**: G1(요금제), I2(부가서비스), D1(할인)에 따라 추가 체크 수행

### 7.2 확장성
- 새로운 상품구분 추가 시 `b000_main_proc`에 분기 추가
- 각 상품구분별 체크 함수는 독립적으로 확장 가능

### 7.3 성능 고려사항
- DB 조회 최소화를 위해 필요한 경우에만 조회
- 조건부 조회: 제한 플래그가 "Y"인 경우에만 상세 조회 수행
