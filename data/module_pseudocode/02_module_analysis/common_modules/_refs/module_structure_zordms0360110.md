### 3.3 부가서비스_가입체크 (가입 체크)

**설명**: 부가서비스 가입 시 모든 조건을 체크

**입력 파라미터**:
- 상품 객체 (상품ID, 가입해지구분코드 등)

**주요 처리 로직**:

#### 1단계: 상품별 특화 Rule 처리
```
IF 상품ID LIKE "NA%" THEN
    상품별_가입Rule_처리(상품)
END IF
```
- 상품ID가 "NA"로 시작하는 경우 개별 상품 특화 규칙 적용
- 수백 개의 상품별 세부 규칙 존재

#### 2단계: 공통 가입 조건 체크

**2-1. 중복 가입 체크**
- 고객가입상품 테이블 조회
- 기존가입.상태 = "가입중" → 처리 불가
- 불가사유: "이미 가입한 상품입니다"

**2-2. 기준 요금제 체크**
- 부가서비스기준요금제 테이블 조회
- 기준요금제목록이 존재하는 경우
- 현재요금제 ∉ 기준요금제목록 → 처리 불가
- 불가사유: "기준 요금제를 먼저 가입해주세요"

**2-3. 선행 부가서비스 체크**
- 선행부가서비스 테이블 조회
- 모든 선행서비스의 가입 여부 확인
- 미가입 서비스 존재 → 처리 불가
- 불가사유: "[선행서비스명] 먼저 가입 필요"

**2-4. 배타 서비스 체크**
- 배타부가서비스 테이블 조회
- 배타 관계 서비스 가입 여부 확인
- 가입중인 배타 서비스 존재 → 처리 불가
- 불가사유: "[배타서비스명]과 동시 가입 불가"

**2-5. 나이 제한 체크**
- 상품정보 테이블 조회
- 최소나이 > 0: 고객나이 < 최소나이 → 처리 불가
- 최대나이 > 0: 고객나이 > 최대나이 → 처리 불가

**2-6. 단말 제한 체크**
- 단말제한 = "Y"인 경우
- 고객현재단말 조회
- 단말속성 조회 (단말모델코드, 상품ID)
- 단말속성.지원여부 = "N" → 처리 불가
- 불가사유: "지원하지 않는 단말입니다"

**반환값**:
- 출력.처리가능여부: "Y" / "N"
- 출력.불가사유: 처리 불가 시 구체적 사유

### 3.4 부가서비스_해지체크 (해지 체크)

**설명**: 부가서비스 해지 시 모든 조건을 체크

**주요 처리 로직**:

#### 1. 가입 여부 체크
- 고객가입상품 테이블 조회
- 기존가입 = NOT_FOUND OR 기존가입.상태 = "해지" → 처리 불가
- 불가사유: "가입하지 않은 상품입니다"

#### 2. 최소 사용 기간 체크
- 기존가입.최소사용기간 > 0인 경우
- 사용기간 = 계산(기존가입.가입일, 입력.처리일자)
- 사용기간 < 최소사용기간 → 처리 불가
- 불가사유: "최소 [N]일 이상 사용 필요"

#### 3. 종속 서비스 체크
- 종속부가서비스 테이블 조회
- 현재 서비스를 필수로 하는 다른 서비스 확인
- 종속서비스.상태 = "가입중" → 처리 불가
- 불가사유: "[종속서비스명] 먼저 해지 필요"

#### 4. 약정 체크
- 기존가입.약정여부 = "Y"인 경우
- 상품약정정보 테이블 조회
- 약정정보.약정상태 = "유효" → 위약금 계산
- 출력.위약금 설정
- 출력.경고메시지: "위약금 [금액]원 발생"

#### 5. 프로모션 혜택 체크
- 상품프로모션 테이블 조회
- 프로모션정보.상태 = "유효" → 혜택반환금액 계산
- 출력.혜택반환금액 설정
- 출력.경고메시지: "프로모션 혜택 반환 [금액]원"

**반환값**:
- 출력.처리가능여부: "Y" / "N"
- 출력.위약금: 발생하는 위약금
- 출력.혜택반환금액: 반환해야 할 프로모션 혜택
- 출력.경고메시지: 위약금/혜택반환 안내

### 3.8 계산위약금 (위약금 계산)

**설명**: 약정 위약금을 계산

**입력 파라미터**:
- 약정정보 객체

**처리 로직**:
1. 경과개월수 계산 (약정시작일 ~ 현재일자)
2. 남은개월수 계산 (약정개월수 - 경과개월수)
3. 남은개월수 ≤ 0 → 위약금 = 0
4. 경과개월수에 따른 위약금 계산
   - 경과개월수 < 12: 위약금 = 기본위약금
   - 경과개월수 ≥ 12: 위약금 = 기본위약금 × 0.5

**반환값**: 위약금 금액

---

### 3.9 계산혜택반환금액 (프로모션 혜택 반환 계산)

**설명**: 프로모션 혜택 반환 금액을 계산

**입력 파라미터**:
- 프로모션정보 객체

**처리 로직**:
1. 경과개월수 계산 (프로모션시작일 ~ 현재일자)
2. 경과개월수 ≥ 최소사용개월수 → 반환금액 = 0
3. 반환비율 계산
   ```
   반환비율 = (최소사용개월수 - 경과개월수) / 최소사용개월수
   ```
4. 반환금액 계산
   ```
   반환금액 = 프로모션혜택금액 × 반환비율
   ```

**반환값**: 반환금액

| **고객현재단말** | 현재 사용 단말 정보 | 부가서비스_가입체크, 상품별_가입Rule |
| **단말속성** | 단말 지원 여부 확인 | 부가서비스_가입체크, 상품별_가입Rule |
| **종속부가서비스** | 종속 서비스 목록 조회 | 부가서비스_해지체크 |
| **상품약정정보** | 약정 정보 및 위약금 계산 | 부가서비스_해지체크, 상품별_가입Rule |
| **상품프로모션** | 프로모션 정보 및 혜택 반환 | 부가서비스_해지체크 |
| **고객약정정보** | 고객 약정 정보 조회 | 상품별_가입Rule |

### 5.2 조회 패턴

**가입 체크 시**:
```
1. 고객가입상품(서비스관리번호, 상품ID) → 중복 체크
2. 부가서비스기준요금제(상품ID) → 기준 요금제 목록
3. 고객현재요금제(서비스관리번호) → 현재 요금제
4. 선행부가서비스(상품ID) → 선행 서비스 목록
5. 배타부가서비스(상품ID) → 배타 서비스 목록
6. 단말속성(단말모델코드, 상품ID) → 단말 지원 여부
```

**해지 체크 시**:
```
1. 고객가입상품(서비스관리번호, 상품ID) → 가입 여부
2. 종속부가서비스(상품ID, 서비스관리번호) → 종속 서비스
3. 상품약정정보(서비스관리번호, 상품ID) → 약정 정보
4. 상품프로모션(서비스관리번호, 상품ID) → 프로모션 정보
```

## 7. 특이사항 및 주의사항

### 7.1 대용량 모듈 특성
- **27,000줄 이상**: 수백 개의 상품별 특화 규칙으로 인한 대용량
- **상품별 분기**: 각 상품ID마다 고유한 가입 규칙 존재
- **유지보수 복잡도**: 신규 상품 추가 시 개별 함수 추가 필요

### 7.2 처리 특성
- **2단계 체크**: 상품별 특화 Rule → 공통 조건 체크
- **조기 종료**: 각 체크 단계에서 조건 미충족 시 즉시 실패 반환
- **가입/해지 분리**: 가입과 해지 로직이 완전히 분리되어 처리

### 7.3 확장성 고려사항
- **신규 상품 추가**: `상품별_가입Rule_처리` 함수에 새로운 분기 추가
- **공통 규칙 변경**: 공통 가입/해지 조건 체크 로직 수정
- **성능 최적화**: 상품별 분기를 룰 엔진으로 전환 고려

### 7.4 위약금 및 혜택 반환 계산
- **위약금**: 경과개월수에 따라 차등 적용 (12개월 기준 50% 감소)
- **혜택 반환**: 최소사용기간 대비 경과기간 비율로 계산
- **경고 메시지**: 금액 발생 시 명확한 안내 메시지 제공
