### 3.4 c000_상품공통기능체크 (상품 공통 체크)

**설명**: 모든 상품에 공통으로 적용되는 기본 체크 로직

**입력 파라미터**:
- 상품 객체 (상품ID, 상품구분코드 등)

**주요 처리 로직**:

#### 1. 상품 기본정보 조회
- 상품마스터 테이블에서 상품 정보 조회
- 조회 키: 상품.상품ID, 입력.처리일시
- 상품 미존재 시:
  - 출력.처리가능여부 ← "N"
  - 출력.불가사유 ← "상품정보 없음"
  - RETURN 실패

#### 2. 판매 가능 여부 체크
- 상품정보.판매가능여부 = "N" 확인
- 판매 중지된 경우:
  - 출력.처리가능여부 ← "N"
  - 출력.불가사유 ← "판매 중지된 상품"
  - RETURN 실패

#### 3. 판매 기간 체크
- 처리일자가 판매시작일 ~ 판매종료일 범위 내인지 확인
- 범위 밖인 경우:
  - 출력.처리가능여부 ← "N"
  - 출력.불가사유 ← "판매 기간 아님"
  - RETURN 실패

#### 4. 가입 대상 체크 (나이)
- **최소나이 체크**:
  - 상품정보.최소나이 > 0 AND 고객.나이 < 최소나이
  - 미달 시: "최소 가입 나이 미달"
  
- **최대나이 체크**:
  - 상품정보.최대나이 > 0 AND 고객.나이 > 최대나이
  - 초과 시: "최대 가입 나이 초과"

#### 5. 채널 제한 체크
- 상품정보.판매채널제한 = "Y"인 경우:
  - 상품판매채널 테이블에서 허용채널목록 조회
  - 입력.채널이 허용채널목록에 없으면:
    - 출력.불가사유 ← "해당 채널에서 판매 불가"
    - RETURN 실패

#### 6. 중복 가입 체크
- 상품정보.중복가입불가 = "Y"인 경우:
  - 고객가입상품 테이블 조회 (고객.서비스관리번호, 상품.상품ID)
  - 기존가입 EXISTS AND 기존가입.상태 = "가입중":
    - 출력.불가사유 ← "이미 가입한 상품"
    - RETURN 실패

#### 7. 재고 체크
- 상품정보.재고관리 = "Y"인 경우:
  - 상품재고 테이블 조회 (상품.상품ID, 입력.처리일자)
  - 재고정보.재고수량 ≤ 0:
    - 출력.불가사유 ← "재고 부족"
    - RETURN 실패

#### 8. 블랙리스트 체크
- 고객.블랙리스트여부 = "Y" 확인
- 블랙리스트인 경우:
  - 출력.불가사유 ← "가입 제한 고객"
  - RETURN 실패

**반환값**:
- 출력.처리가능여부: "Y" / "N"
- 출력.불가사유: 처리 불가 시 사유

**에러 처리**:
- 각 체크 단계에서 조건 미충족 시 즉시 실패 반환
- 불가사유를 명확히 설정하여 반환

### 3.5 요금제_공통체크 (G1)

**설명**: 요금제 상품에 특화된 체크 로직

**입력 파라미터**:
- 상품 객체 (요금제 정보)

**주요 처리 로직**:

#### 1. 기존 요금제 조회
- 고객현재요금제 테이블 조회 (고객.서비스관리번호)
- 고객의 현재 가입 요금제 확인

#### 2. 요금제 변경 가능 여부 체크
기존요금제가 존재하는 경우:

- **최소 사용 기간 체크**:
  - 기존요금제.최소사용기간 > 0인 경우
  - 사용기간 = 계산(기존요금제.가입일, 입력.처리일자)
  - 사용기간 < 최소사용기간:
    - 출력.불가사유 ← "최소 사용 기간 미달"
    - RETURN 실패

- **요금제 변경 제한 체크**:
  - 요금제변경제한 테이블 조회 (기존요금제.요금제ID, 상품.상품ID)
  - 변경제한 EXISTS AND 변경제한.변경불가 = "Y":
    - 출력.불가사유 ← 변경제한.불가사유
    - RETURN 실패

#### 3. 세대 제한 체크
- 기존요금제.세대 = "3G" AND 상품.세대 = "5G":
  - 직접변경불가 = "Y"인 경우:
    - 출력.불가사유 ← "4G 거쳐서 변경 필요"
    - RETURN 실패
  - **비즈니스 의미**: 3G에서 5G로 직접 변경 불가, 4G 경유 필요

#### 4. 약정 체크
- 고객약정정보 테이블 조회 (고객.서비스관리번호)
- 약정정보 EXISTS AND 약정정보.약정상태 = "유효":
  - 위약금 = 계산위약금(약정정보)
  - 위약금 > 0인 경우:
    - 출력.위약금 ← 위약금
    - 출력.경고메시지 ← "약정 위약금 발생"
  - **주의**: 위약금이 있어도 처리는 가능 (경고만 표시)

**반환값**:
- 출력.처리가능여부: "Y" / "N"
- 출력.위약금: 발생하는 위약금 금액
- 출력.경고메시지: 약정 위약금 발생 안내

### 3.6 부가서비스_공통체크 (I2)

**설명**: 부가서비스 상품에 특화된 체크 로직

**입력 파라미터**:
- 상품 객체 (부가서비스 정보)

**주요 처리 로직**:

#### 1. 기준 요금제 가입 여부 체크
- 상품.기준요금제필수 = "Y"인 경우:
  - 상품기준요금제 테이블 조회 (상품.상품ID)
  - 고객현재요금제 테이블 조회 (고객.서비스관리번호)
  - 현재요금제.요금제ID NOT IN 기준요금제목록:
    - 출력.불가사유 ← "기준 요금제 미가입"
    - RETURN 실패
  - **비즈니스 의미**: 특정 요금제 가입자만 이용 가능한 부가서비스

#### 2. 선행 부가서비스 체크
- 상품.선행서비스필수 = "Y"인 경우:
  - 선행부가서비스 테이블 조회 (상품.상품ID)
  - 각 선행서비스에 대해:
    - 고객가입여부 테이블 조회 (고객.서비스관리번호, 선행서비스.상품ID)
    - 가입여부 = "미가입":
      - 출력.불가사유 ← "선행 서비스 미가입: " + 선행서비스.상품명
      - RETURN 실패
  - **비즈니스 의미**: 순차적 가입이 필요한 서비스 (예: 기본 → 프리미엄)

#### 3. 배타 관계 체크
- 배타부가서비스 테이블 조회 (상품.상품ID)
- 각 배타서비스에 대해:
  - 고객가입여부 테이블 조회 (고객.서비스관리번호, 배타서비스.상품ID)
  - 가입여부 = "가입중":
    - 출력.불가사유 ← "배타 서비스 가입중: " + 배타서비스.상품명
    - RETURN 실패
- **비즈니스 의미**: 동시 가입 불가능한 서비스 (예: A플랜 ↔ B플랜)

#### 4. 단말 제한 체크
- 상품.단말제한 = "Y"인 경우:
  - 고객현재단말 테이블 조회 (고객.서비스관리번호)
  - 상품허용단말 테이블 조회 (상품.상품ID)
  - 단말정보.단말모델코드 NOT IN 허용단말목록:
    - 출력.불가사유 ← "지원하지 않는 단말"
    - RETURN 실패
  - **비즈니스 의미**: 특정 단말에서만 사용 가능한 서비스

**반환값**:
- 출력.처리가능여부: "Y" / "N"
- 출력.불가사유: 처리 불가 시 구체적 사유
