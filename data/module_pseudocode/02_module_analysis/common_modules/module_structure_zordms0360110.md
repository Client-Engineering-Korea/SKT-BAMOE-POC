### 3.4 부가서비스_가입체크 (가입 체크)

**설명**: 부가서비스 가입 시 상품별 특화 규칙과 공통 조건을 모두 체크

**입력 파라미터**:
- 상품 객체 (상품ID, 가입해지구분코드 등)

**주요 처리 로직**:

#### 1단계: 상품별 특화 Rule 처리
```
IF 상품ID LIKE "NA%" THEN
    상품별_가입Rule_처리(상품)
END IF
```
- 상품ID가 "NA"로 시작하는 경우 개별 상품 특화 규칙 적용
- 수백 개의 상품별 세부 규칙 존재 (통화중대기, 데이터쉐어링, 선택약정할인, T지원금약정 등)

#### 2단계: 공통 가입 조건 체크

**2-1. 중복 가입 체크**
- 고객가입상품 테이블 조회 (고객.서비스관리번호, 상품ID)
- 조건: 기존가입 EXISTS AND 기존가입.상태 = "가입중"
- 결과: 처리 불가
- 불가사유: "이미 가입한 상품입니다"

**2-2. 기준 요금제 체크**
- 부가서비스기준요금제 테이블 조회 (상품ID)
- 기준요금제목록.건수 > 0인 경우
  - 고객현재요금제 테이블 조회 (고객.서비스관리번호)
  - 조건: 현재요금제.요금제ID NOT IN 기준요금제목록
  - 결과: 처리 불가
  - 불가사유: "기준 요금제를 먼저 가입해주세요"

**2-3. 선행 부가서비스 체크**
- 선행부가서비스 테이블 조회 (상품ID)
- FOR EACH 선행서비스
  - 고객가입여부 조회 (고객.서비스관리번호, 선행서비스.상품ID)
  - 조건: 가입여부 = "미가입"
  - 결과: 처리 불가
  - 불가사유: "[선행서비스.상품명] 먼저 가입 필요"

**2-4. 배타 서비스 체크**
- 배타부가서비스 테이블 조회 (상품ID)
- FOR EACH 배타서비스
  - 고객가입여부 조회 (고객.서비스관리번호, 배타서비스.상품ID)
  - 조건: 가입여부 = "가입중"
  - 결과: 처리 불가
  - 불가사유: "[배타서비스.상품명]과 동시 가입 불가"

**2-5. 나이 제한 체크**
- 상품정보 테이블 조회 (상품ID)
- 최소나이 체크
  - 조건: 상품정보.최소나이 > 0 AND 고객.나이 < 상품정보.최소나이
  - 결과: 처리 불가
  - 불가사유: "최소 [최소나이]세 이상 가입 가능"
- 최대나이 체크
  - 조건: 상품정보.최대나이 > 0 AND 고객.나이 > 상품정보.최대나이
  - 결과: 처리 불가
  - 불가사유: "최대 [최대나이]세 이하 가입 가능"

**2-6. 단말 제한 체크**
- 조건: 상품정보.단말제한 = "Y"
- 고객현재단말 테이블 조회 (고객.서비스관리번호)
- 단말속성 테이블 조회 (단말정보.단말모델코드, 상품ID)
- 조건: 단말속성.지원여부 = "N"
- 결과: 처리 불가
- 불가사유: "지원하지 않는 단말입니다"

**반환값**:
- 출력.처리가능여부: "Y" / "N"
- 출력.불가사유: 처리 불가 시 구체적 사유

**에러 처리**:
- 각 체크 단계에서 조건 미충족 시 즉시 실패 반환
- 불가사유를 명확히 설정하여 반환

### 3.5 부가서비스_해지체크 (해지 체크)

**설명**: 부가서비스 해지 시 모든 조건을 체크하고 위약금 및 혜택 반환 금액을 계산

**입력 파라미터**:
- 상품 객체 (상품ID 등)

**주요 처리 로직**:

#### 1. 가입 여부 체크
- 고객가입상품 테이블 조회 (고객.서비스관리번호, 상품.상품ID)
- 조건: 기존가입 = NOT_FOUND OR 기존가입.상태 = "해지"
- 결과: 처리 불가
- 불가사유: "가입하지 않은 상품입니다"

#### 2. 최소 사용 기간 체크
- 조건: 기존가입.최소사용기간 > 0
- 사용기간 계산 (기존가입.가입일 ~ 입력.처리일자)
- 조건: 사용기간 < 기존가입.최소사용기간
- 결과: 처리 불가
- 불가사유: "최소 [최소사용기간]일 이상 사용 필요"

#### 3. 종속 서비스 체크
- 종속부가서비스 테이블 조회 (상품.상품ID, 고객.서비스관리번호)
- 현재 서비스를 필수로 하는 다른 서비스 확인
- FOR EACH 종속서비스
  - 조건: 종속서비스.상태 = "가입중"
  - 결과: 처리 불가
  - 불가사유: "[종속서비스.상품명] 먼저 해지 필요"

#### 4. 약정 체크
- 조건: 기존가입.약정여부 = "Y"
- 상품약정정보 테이블 조회 (고객.서비스관리번호, 상품.상품ID)
- 조건: 약정정보 EXISTS AND 약정정보.약정상태 = "유효"
- 위약금 계산 (`계산위약금(약정정보)`)
- 조건: 위약금 > 0
  - 출력.위약금 설정
  - 출력.경고메시지: "위약금 [위약금]원 발생"

#### 5. 프로모션 혜택 체크
- 상품프로모션 테이블 조회 (고객.서비스관리번호, 상품.상품ID)
- 조건: 프로모션정보 EXISTS AND 프로모션정보.상태 = "유효"
- 혜택반환금액 계산 (`계산혜택반환금액(프로모션정보)`)
- 조건: 혜택반환금액 > 0
  - 출력.혜택반환금액 설정
  - 출력.경고메시지: "프로모션 혜택 반환 [혜택반환금액]원"

**반환값**:
- 출력.처리가능여부: "Y" / "N"
- 출력.위약금: 발생하는 위약금 (있는 경우)
- 출력.혜택반환금액: 반환해야 할 프로모션 혜택 (있는 경우)
- 출력.경고메시지: 위약금/혜택반환 안내

### 3.9 계산위약금 (위약금 계산)

**설명**: 약정 위약금을 경과개월수에 따라 차등 계산

**입력 파라미터**:
- 약정정보 객체 (약정시작일, 약정개월수, 기본위약금 등)

**주요 처리 로직**:

1. **경과개월수 계산**
   - 경과개월수 = 계산(약정정보.약정시작일, 현재일자)

2. **남은개월수 계산**
   - 남은개월수 = 약정정보.약정개월수 - 경과개월수
   - 조건: 남은개월수 ≤ 0
   - 결과: 위약금 = 0 반환

3. **경과개월수에 따른 위약금 계산**
   - 경과개월수 < 12개월
     - 위약금 = 약정정보.기본위약금 × 1.0 (100%)
   - 경과개월수 ≥ 12개월
     - 위약금 = 약정정보.기본위약금 × 0.5 (50%)

**반환값**: 위약금 금액

---

### 3.10 계산혜택반환금액 (프로모션 혜택 반환 계산)

**설명**: 프로모션 혜택 반환 금액을 최소사용기간 대비 경과기간 비율로 계산

**입력 파라미터**:
- 프로모션정보 객체 (시작일, 최소사용개월수, 혜택금액 등)

**주요 처리 로직**:

1. **경과개월수 계산**
   - 경과개월수 = 계산(프로모션정보.시작일, 현재일자)

2. **최소사용개월수 충족 여부 확인**
   - 조건: 경과개월수 ≥ 프로모션정보.최소사용개월수
   - 결과: 반환금액 = 0 반환

3. **반환비율 계산**
   ```
   반환비율 = (최소사용개월수 - 경과개월수) / 최소사용개월수
   ```

4. **반환금액 계산**
   ```
   반환금액 = 프로모션정보.혜택금액 × 반환비율
   ```

**반환값**: 반환금액

## 7. 특이사항 및 주의사항

### 7.1 대용량 모듈 특성
- **27,000줄 이상**: 수백 개의 상품별 특화 규칙으로 인한 대용량
- **상품별 분기**: 각 상품ID마다 고유한 가입 규칙 존재
- **유지보수 복잡도**: 신규 상품 추가 시 개별 함수 추가 필요
- **코드 중복**: 유사한 체크 로직이 상품별로 반복됨

### 7.2 처리 특성
- **2단계 체크 구조**: 상품별 특화 Rule → 공통 조건 체크
- **조기 종료 패턴**: 각 체크 단계에서 조건 미충족 시 즉시 실패 반환
- **가입/해지 분리**: 가입과 해지 로직이 완전히 분리되어 처리
- **순차 처리**: 상품 목록을 순회하며 각각 독립적으로 체크

### 7.3 확장성 고려사항
- **신규 상품 추가**
  - `상품별_가입Rule_처리` 함수에 새로운 IF 분기 추가
  - 상품별 특화 함수 구현 (예: `신규상품_가입Rule`)
  - 관련 DB 테이블 및 조회 로직 추가

- **공통 규칙 변경**
  - 공통 가입/해지 조건 체크 로직 수정
  - 모든 상품에 일괄 적용됨

- **룰 엔진 전환 고려**
  - 현재: 하드코딩된 IF-ELSE 분기
  - 개선: 룰 엔진 (Drools 등)으로 전환하여 유지보수성 향상
  - 상품별 규칙을 외부 설정으로 관리

### 7.4 성능 최적화 포인트
- **조건부 조회**: 필요한 경우에만 DB 조회 수행
  - 예: 기준요금제목록.건수 > 0인 경우에만 현재요금제 조회
  - 예: 단말제한 = "Y"인 경우에만 단말속성 조회

- **캐싱 전략**
  - 상품정보, 단말속성 등 자주 조회되는 마스터 데이터 캐싱
  - 고객가입상품 등 트랜잭션 데이터는 실시간 조회

- **배치 조회**
  - 선행서비스, 배타서비스 목록을 한 번에 조회
  - FOR EACH 루프 내 개별 조회 최소화

### 7.5 위약금 및 혜택 반환 계산
- **위약금 계산 로직**
  - 경과개월수 기반 차등 적용
  - 12개월 기준: 100% → 50%
  - 단순하지만 명확한 계산식

- **혜택 반환 계산 로직**
  - 최소사용기간 대비 경과기간 비율
  - 비율 기반 계산으로 공정성 확보
  - 반환비율 = (최소사용개월수 - 경과개월수) / 최소사용개월수

- **경고 메시지**
  - 금액 발생 시 명확한 안내 메시지 제공
  - 고객 의사결정 지원

### 7.6 데이터 정합성
- **중복 가입 방지**: 동일 상품 중복 가입 철저히 차단
- **선행/종속 관계**: 서비스 간 의존성 관리
- **배타 관계**: 동시 가입 불가 서비스 관리
- **약정 정보**: 위약금 계산의 정확성 보장

### 7.7 에러 처리 전략
- **명확한 불가사유**: 각 체크 실패 시 구체적인 사유 제공

- **경고 vs 불가**: 위약금은 경고, 조건 미충족은 불가
