## 3. 주요 함수 상세

### 3.1 zordmcom01040 (메인 함수)

**설명**: 공통 코드의 존재 여부와 유효성을 검증하는 핵심 함수

**입력 파라미터**:
- 입력.코드그룹
- 입력.코드값

**주요 처리 로직**:

1. **공통코드 조회** (line 8)
   - DB조회("공통코드테이블", 입력.코드그룹, 입력.코드값)

2. **코드 존재 여부 체크** (line 10-15)
   - IF 공통코드 = NOT_FOUND
   - 출력.코드존재여부 ← "N"
   - 출력.유효여부 ← "N"
   - RETURN 성공

3. **기본 정보 설정** (line 17-19)
   - 출력.코드존재여부 ← "Y"
   - 출력.코드명 ← 공통코드.코드명
   - 출력.코드설명 ← 공통코드.코드설명

4. **유효기간 체크** (line 21-25)
   - IF 현재일자 < 공통코드.시작일 OR 현재일자 > 공통코드.종료일
   - 출력.유효여부 ← "N"
   - 출력.메시지 ← "유효기간 아님"

5. **사용 여부 체크** (line 27-31)
   - IF 공통코드.사용여부 = "N"
   - 출력.유효여부 ← "N"
   - 출력.메시지 ← "사용 중지된 코드"

6. **유효한 코드 처리** (line 33-37)
   - 출력.유효여부 ← "Y"
   - 추가속성 1, 2, 3 설정

**반환값**: 성공 (항상)

**특징**:
- 코드가 없어도 오류가 아닌 성공 반환
- 유효여부 플래그로 상태 전달

### 3.2 코드값변환 (유틸리티 함수)

**설명**: 코드값을 코드명으로 변환하는 편의 함수

**입력 파라미터**:
- 코드그룹
- 코드값

**주요 처리 로직**:

1. **코드 정보 조회** (line 45)
   - 코드정보 ← zordmcom01040(코드그룹, 코드값)

2. **유효성 확인 및 반환** (line 47-51)
   - IF 코드정보.유효여부 = "Y"
   - RETURN 코드정보.코드명
   - ELSE RETURN "알 수 없음"

**반환값**: 코드명 또는 "알 수 없음"

**사용 예시**:
```
채널명 ← 코드값변환("채널구분", "01")
// 반환: "온라인" 또는 "알 수 없음"
```

## 7. 특이사항 및 주의사항

### 7.1 처리 특성

**1. 오류가 아닌 상태 반환**
- 코드가 없어도 오류 발생하지 않음
- 유효여부 플래그로 상태 전달
- 호출자가 유효여부를 확인하여 처리

**2. 다단계 검증**
- 존재 여부 → 유효기간 → 사용 여부 순차 체크
- 각 단계에서 실패 시 즉시 반환
- 모든 체크 통과 시에만 유효

**3. 추가 속성 지원**
- 속성1, 2, 3으로 확장 가능
- 코드별 추가 정보 저장 가능

### 7.2 확장성 고려사항

**1. 코드 그룹 추가**
- 공통코드테이블에 새로운 코드그룹 추가
- 함수 수정 없이 확장 가능

**2. 속성 확장**
- 현재 3개 속성 지원
- 필요 시 속성4, 5 추가 가능

**3. 유효성 규칙 추가**
```
IF 공통코드.권한레벨 > 사용자.권한레벨 THEN
    출력.유효여부 ← "N"
    출력.메시지 ← "권한 부족"
END IF
```

### 7.3 성능 최적화 포인트

**1. 캐싱 전략**
- 공통코드는 변경 빈도 낮음
- 메모리 캐싱 적용 권장
- 캐시 TTL: 1시간 ~ 1일

**2. 인덱스 최적화**
- 공통코드테이블 인덱스: (코드그룹, 코드값)
- 복합 인덱스로 조회 성능 향상

**3. 배치 조회**
```
// 여러 코드를 한 번에 조회
코드목록 ← DB조회("공통코드테이블", 
                 WHERE 코드그룹 IN (그룹목록))
```

### 7.4 주의사항

**1. 유효기간 관리**
- 시작일, 종료일 정확히 설정
- 종료일 경과 코드는 자동 무효화
- 정기적인 유효기간 점검 필요

**2. 사용여부 플래그**
- 코드 삭제 대신 사용여부 = "N" 설정
- 이력 데이터 보존
- 재활성화 가능

**3. 코드값변환 함수 사용**
- 화면 표시용으로 적합
- 유효하지 않은 코드는 "알 수 없음" 표시
- 비즈니스 로직에서는 zordmcom01040 직접 호출 권장

**4. NULL 처리**
- 코드그룹, 코드값 NULL 체크 필요
- 입력 검증 로직 추가 권장
